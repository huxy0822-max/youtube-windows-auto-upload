# 统一控制台说明

以后日常操作，优先只用这两个入口：

- `dashboard.py`
- `启动统一控制台.bat`

## 这版控制台的核心逻辑

现在控制台里有 3 块能力，但都统一从 `快捷开始` 发车：

1. 渲染
2. 上传
3. 标题 / 简介 / 缩略图配置

其中“上传”又分成 3 种模式，直接在 `快捷开始 -> 上传方式` 里选：

- `单频道上传`
  适合：一个频道单独发
  设置页：`上传`
- `多赛道任务清单`
  适合：今天要跑多个 `tag`
  设置页：还是 `上传`，但实际会按 `快捷开始` 里的任务清单跑
- `同分组现成视频`
  适合：一个文件夹里有多条现成视频，要按顺序发到同一个比特浏览器分组下的多个频道
  设置页：`分组批量上传`

重点变化：

- 以后不需要记 `batch_upload.py / bulk_upload.py / group_upload_batch.py` 这些脚本名
- 先在 `快捷开始` 选上传方式
- 再打开对应设置页检查参数
- 最后回到 `快捷开始` 点按钮

## 快捷开始怎么用

`快捷开始` 现在是总入口，建议按这个顺序：

1. 选 `tag`
2. 选日期
3. 决定今天开不开 `渲染模块 / 上传模块`
4. 在 `上传方式` 里选今天要用哪条上传链路
5. 需要的话去 `提示词` / `当日内容` / 对应上传设置页补参数
6. 回来点 `开始当前流程`

如果你只想直接跑上传，不想跑渲染：

- 直接点 `仅跑当前上传方式`

如果你只想先把视频做出来，不上传：

- 关闭 `上传模块`
- 点 `开始当前流程`
- 或直接点 `仅渲染`

## 三种上传模式分别怎么理解

### 1. 单频道上传

适合：

- 今天只发一个频道

需要检查：

- `上传` 页里的 `tag`
- `频道`
- 当天 `generation_map.json` 对应的标题 / 简介 / 缩略图

### 2. 多赛道任务清单

适合：

- 今天要跑多个赛道
- 每个赛道还是走原来的素材目录和输出目录结构

需要检查：

- `快捷开始` 里的多赛道任务清单
- `路径配置` 里的根目录
- 每个赛道的 `workspace/music/<tag>/`
- 每个赛道的 `workspace/base_image/<tag>/`

### 3. 同分组现成视频

适合：

- 你已经有成品视频
- 一个文件夹里放了多条视频
- 想把这些视频按顺序分给同一个比特浏览器分组下的多个频道

需要检查：

- `分组批量上传` 页里的分组 `tag`
- `现成视频文件夹`
- 可选的缩略图文件夹
- 是否儿童内容
- 是否 AI / 合成内容
- 分类
- 是否定时发布

注意：

- 这个模式是“上传现成视频”的链路
- 它不参与渲染
- 所以如果你选了这个模式，就不要同时开 `渲染模块`

## 提示词和当日内容放哪里

### 提示词

在 `提示词` 页维护：

- 文本模型
- 图片模型
- 主提示词
- 标题库
- 输出语言
- 风格 / 受众 / 角度

### 当日内容

在 `当日内容` 页维护：

- 标题
- 简介
- 封面文件名
- AB 标题

保存后可以同步 `upload_manifest.json`，避免上传读到旧内容。

## 推荐日常流程

### 场景 A：一堆音频 + 图片，要先做视频再上传

1. 去 `路径配置` 确认目录
2. 去 `提示词` 检查模板
3. 去 `当日内容` 检查标题 / 简介 / 封面
4. 回 `快捷开始`
5. 打开 `渲染模块`
6. 打开 `上传模块`
7. 选择 `上传方式`
8. 点 `开始当前流程`

### 场景 B：视频已经做好了，只想上传

1. 回 `快捷开始`
2. 关闭 `渲染模块`
3. 打开 `上传模块`
4. 选择正确的 `上传方式`
5. 点 `仅跑当前上传方式`

### 场景 C：一个文件夹里 5 个成品视频，要发到同一分组 5 个频道

1. 去 `快捷开始`
2. 关闭 `渲染模块`
3. 打开 `上传模块`
4. `上传方式` 选 `同分组现成视频`
5. 点 `打开当前上传设置页`
6. 在 `分组批量上传` 页选择分组、视频文件夹、上传选项
7. 先点 `预览并准备上传计划`
8. 确认没问题后点 `开始分组批量上传`

## 还保留的旧入口

旧界面还在，但不建议日常优先用：

- `app.py`
- `scheduler_gui.py`

保留它们只是为了兼容旧习惯，不代表现在的主流程还应该从那里开始。

## 新主入口：按窗口任务上传

这版开始，`上传` 页不再只是单频道按钮，而是改成“窗口任务页”。

推荐顺序：

1. 先在 `上传` 页选默认分组和日期。
2. 再选范围模式：
   `同一分组的一批窗口` / `多个分组，各自列窗口` / `逐窗口单独配置`
3. 再设默认发布规则：
   可见性、分类、儿童内容、AI 内容、定时发布时间
4. 点 `预览今日窗口任务`
5. 确认预览无误后，点 `开始按窗口任务上传`

### 三种范围模式怎么填

#### 1. 同一分组的一批窗口

直接在 `同组窗口号` 输入框里填：

```text
90,91,92
```

适合：

- 今天都是同一个 BitBrowser 分组
- 只是要精确到这个分组里的哪几个窗口

#### 2. 多个分组，各自列窗口

在 `多分组输入区` 每行写一组：

```text
面壁者: 90,91
芝加哥蓝调: 94,95
```

适合：

- 今天要同时上传多个分组
- 但每个分组只跑其中几个窗口

#### 3. 逐窗口单独配置

在 `逐窗口覆盖输入区` 每行写：

```text
90 | 面壁者
91 | 面壁者 | visibility=private | category=Music
95 | 芝加哥蓝调 | title=自定义标题 | altered_content=true
```

适合：

- 每个窗口规则都可能不一样
- 某些窗口要单独改标题、描述、标签、分类、可见性

### 逐窗口模式支持哪些覆盖字段

可用字段：

- `visibility`
- `category`
- `made_for_kids`
- `altered_content`
- `scheduled_publish_at`
- `title`
- `description`
- `tag_list`
- `thumbnails`
- `ab_titles`

说明：

- 如果某个窗口没写这些字段，就会走 `上传` 页里的默认发布规则
- 如果写了，就以该窗口自己的覆盖值为准

### 和快捷开始的关系

`快捷开始` 现在会先让你决定：

1. 今天是不是走 `按窗口任务上传`
2. 如果是，属于哪一种范围模式
3. 然后再去决定要不要同时启用渲染

也就是说：

- `快捷开始` 负责先问清楚“今天传谁”
- `上传` 页负责把窗口任务和默认规则填完整
