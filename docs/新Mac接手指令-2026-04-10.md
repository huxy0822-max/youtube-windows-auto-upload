# 新 Mac 接手指令

这份文档给新 Mac 直接照着做。

## 1. 先装基础环境

如果还没装 Homebrew，先安装它。

然后执行：

```bash
brew install git python ffmpeg
```

## 2. 拉代码并切到当前分支

```bash
cd ~/Desktop
git clone https://github.com/huxy0822-max/youtube-windows-auto-upload.git
cd youtube-windows-auto-upload
git fetch --all --tags
git checkout mac-huxy || git checkout -b mac-huxy origin/mac-huxy
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 3. 生成当前电脑自己的私有配置

```bash
cp scheduler_config.json scheduler_config.local.json
cp config/prompt_studio.json config/prompt_studio.local.json
```

然后只改这两个本机文件：

- `scheduler_config.local.json`
- `config/prompt_studio.local.json`

要填的内容：

- 新 Mac 上真实的音乐目录
- 新 Mac 上真实的底图目录
- 新 Mac 上真实的 metadata 目录
- 新 Mac 上真实的输出目录
- 新 Mac 上真实的长期分组绑定目录
- 真实 API key

不要把真实路径和真实 key 写回公开模板文件。

## 4. 256G Mac 的磁盘规则

这台 Mac 必须这样用：

- `上传后保留天数 = 0`
- 命令行等价参数：`--retain-video-days 0`

当前仓库默认值已经是 `0`。

含义：

- 上传验收成功后，立刻删除本地 `.mp4`
- 只保留 `upload_manifest.json` 和 `upload_report.json`

## 5. 启动控制台

```bash
source .venv/bin/activate
python3 dashboard.py
```

## 6. 第一次让 Codex 接手时直接发这段

```text
这是一个 YouTube 自动化项目。先读 AGENTS.md、README.md、docs/项目功能总纲-2026-03-17.md、docs/新控制台使用说明.md、docs/新Mac接手指令-2026-04-10.md。当前工作分支是 mac-huxy。先检查 scheduler_config.local.json 和 config/prompt_studio.local.json 是否已经填好本机路径与 API key，再带我完成一次最小可运行的流程：1）识别今天要用的分组和素材目录；2）确认上传页窗口设置；3）确认上传后保留天数是 0；4）跑一次最小规模生成/上传/验收；5）确认上传成功后本地视频已经删除。不要回退成 Windows 专属逻辑。
```
