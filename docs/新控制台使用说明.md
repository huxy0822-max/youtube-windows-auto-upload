# 新控制台使用说明

入口：

- `C:\youtube自动化\dashboard.py`
- `C:\youtube自动化\启动统一控制台.bat`

## 1. 先理解整套流程

现在控制台只按这一套逻辑工作：

1. `快捷开始`
   这里只决定今天做什么
2. `上传`
   这里只决定今天哪些窗口要工作
3. `提示词`
   这里只管 API 模板和内容模板
4. `当日内容`
   这里只管某个频道某一天的标题/简介/封面/A-B 标题
5. `路径配置`
   这里只管全局路径和“分组 -> 素材目录”的长期绑定
6. `日志`
   看实时进度、当前阶段、预计剩余时间

## 2. 快捷开始怎么用

这里只选 3 种模式：

- `本日只剪辑`
- `本日只上传`
- `本日剪辑并上传`

说明：

- `本日只上传` 现在不是直接拿旧清单上传
- 它会先按上传页当前任务重新准备新视频和新文案，再进入上传
- 如果你只是想先试链路，先点 `模拟 1-2 分钟`

## 3. 上传页怎么用

上传页是唯一上传入口。

### 第一步：决定今天哪些窗口要工作

你只需要：

- 选 BitBrowser 分组
- 点窗口按钮
- 哪个窗口要做，就点哪个窗口
- 点一次就把它加入下面的任务区

任务区里：

- 1 个窗口 = 单个上传
- 多个窗口 = 批量上传

### 第二步：决定本次素材从哪里来

默认逻辑：

- 留空 `新增窗口用素材目录覆盖`
- 就走该分组在 `路径配置` 里长期绑定的素材目录

如果你今天临时要换目录：

- 在 `新增窗口用素材目录覆盖` 填新目录
- 这次加入任务区的窗口就走这个目录

### 第三步：决定文案来源

上传页里的文案来源只有两种：

- `提示词那套`
- `原先那套`

含义：

- `提示词那套` = 走你在 `提示词` 页配置的 API 模板 + 内容模板
- `原先那套` = 走 `generation_map.json`

旁边的 3 个开关分别是：

- `生成标题/简介/标签`
- `处理缩略图`
- `同步到当日内容`

### 第四步：设置默认规则

这里是所有窗口共用的默认值：

- 可见性
- 分类
- 儿童内容
- AI 内容
- 定时发布日期
- 定时时间
- 时区
- 定时间隔
- 上传完成后自动关闭窗口

当前时区先固定保留：

- `Asia/Taipei (+08:00)`

### 第五步：给单个窗口临时覆盖

点窗口前，先看“加入窗口时”这一排：

- `YPP`
- `数量(每窗口)`
- `可见性`
- `分类`
- `儿童内容`
- `AI 内容`
- `窗口定时覆盖`

这排值会跟着你点窗口一起写进任务区。

## 4. 提示词页怎么用

这里支持多套模板，不再只有默认模板。

你可以：

- 选一个分组
- 选 API 模板
- 选内容模板
- 把当前表单另存为新模板
- 把模板绑定到分组

还支持：

- `测试文本连通性`
- `测试图片连通性`
- `选择截图`
- `粘贴截图`
- `重新识别`

截图识别结果会自动回填目标受众。

## 5. 当日内容页怎么用

这里用于手动改当天落地内容：

- 标题
- 简介
- 封面文件名
- A/B 标题
- YPP

改完可以直接同步到 `upload_manifest.json`。

## 6. 路径配置页怎么用

这里分两块。

### 全局路径

- 音乐目录
- 底图目录
- metadata 目录
- 输出目录
- FFmpeg
- 已用素材目录
- 上传后保留天数

说明：

- `上传后保留天数 = 0` 表示上传验收成功后立即删除本地视频
- 公开仓库里的 `scheduler_config.json` 只是模板
- 当前电脑的真实路径应保存到 `scheduler_config.local.json`

提示词页同理：

- 公开仓库里的 `config/prompt_studio.json` 只放干净模板
- 当前电脑的真实 API key 应保存到 `config/prompt_studio.local.json`

### 分组绑定素材文件夹

这里是长期绑定关系。

比如：

- `0128-华尔兹 -> F:\学习华尔兹\zd\0128-华尔兹`
- `面壁者 -> F:\你的目录\面壁者`

保存后，下次该分组默认就走这个目录。

## 7. 现在已经修掉的关键问题

- 不再只显示前 8 个窗口
- 上传页任务区已恢复，不会被上面控件挤没
- 运行中会显示当前阶段、当前窗口、进度、耗时、预计剩余
- 同一批素材再次生成时，会避开最近已用标题/简介/标签/缩略图提示词
- `本日只上传` 不再直接复用旧 manifest
- Windows 机器会优先检测可用的 GPU 编码器

## 8. 关于显卡加速

控制台现在会自动选择当前机器能用的编码器。

你这台机器实测是：

- `NVIDIA GeForce RTX 3070`

所以正确路线是：

- `h264_nvenc`

不是：

- `h264_amf`

注意：

- GPU 编码已经接上，不再是纯 CPU 编码
- 但如果特效链很重，整体仍然可能被 CPU 滤镜拖慢
- 日志里会直接打印当前编码器，方便确认
