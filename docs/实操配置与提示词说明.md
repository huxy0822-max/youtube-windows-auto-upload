# YouTube 自动化实操配置说明（Windows + 比特浏览器）

## 0) 速查：你想改什么，就去哪里改

| 你想改什么 | 文件 | 关键字段 / 说明 |
|---|---|---|
| 比特分组对应哪些频道 | `config/upload_config.json` | `tag_to_project.<分组名>.non_ypp_serials / ypp_serials` |
| 上传读哪个分组 | 命令行参数 | `--tag 面壁者` |
| 上传读哪一天 | 命令行参数 | `--date 3.12`（内部会转成 `0312`） |
| 文本 / 图片模型模板 | `config/prompt_studio.json` 或 `dashboard.py -> 提示词` | `apiPresets` |
| 主提示词 / 标题库 / 标题数量等 | `config/prompt_studio.json` 或 `dashboard.py -> 提示词` | `contentTemplates` |
| 标题 | `workspace/base_image/<tag>/generation_map.json` | `channels.<序号>.days.<MMDD>.title` |
| 简介 | 同上 | `channels.<序号>.days.<MMDD>.description` |
| 封面文件名 | 同上 | `channels.<序号>.days.<MMDD>.covers` |
| 封面图片文件 | `workspace/base_image/<tag>/` | `<MMDD>_<serial>_cover_01.png` |
| 底图文件 | `workspace/base_image/<tag>/` | `<MMDD>_<serial>.png` |
| 音频文件 | `workspace/music/<tag>/` | 例如 `output_03.mp3` |
| 多赛道任务清单 | `dashboard.py -> 快捷开始` | 每行一个 `tag` |
| 多赛道批量上传 | `dashboard.py -> 上传` 或 `bulk_upload.py` | 按任务清单顺序依次上传 |

## 1) 上传流程里已固定的策略

`batch_upload.py` 里当前已经按你的要求固定为：

- `Altered content = Yes`
- `Category = Music`

另外，首个 `Next` 现在会等待较久（大文件慢上传时，YouTube 早期不显示 `Next`）。

## 2) 标题/简介提示词怎么改

现在本地已经有一个统一入口：

- `dashboard.py -> 提示词`

这个页面参考了你网页版站点的字段，能直接维护：

- 文本 API 模板：`provider / baseUrl / model / temperature / maxTokens`
- 图片 API 模板：`imageBaseUrl / imageModel / imageConcurrency`
- 内容模板：`musicGenre / angle / audience / outputLanguage`
- 生成参数：`titleCount / descCount / thumbCount / titleMin / titleMax / descLen / tagRange`
- 主提示词：`masterPrompt`
- 标题库：`titleLibrary`

先明确：当前仓库上传脚本仍然不“在线生成文案”，它只读现成结果。

- 上传实际读取：`workspace/base_image/<tag>/generation_map.json`
- 你可以直接手改 `title/description`

如果你有外部文案生成器（GPT/Gemini/Claude 等）：

1. 在 `dashboard.py -> 提示词` 改模板、模型名、主提示词、标题库
2. 把结果写回 `generation_map.json`
3. 再执行上传脚本

## 3) 图片提示词怎么改

同理，当前仓库上传阶段不出图，只消费已有图片文件。

- 你要改“图片提示词”，现在优先去 `dashboard.py -> 提示词` 改对应内容模板和图片模型计划
- 出图后把文件覆盖到：
  - 底图：`workspace/base_image/<tag>/<MMDD>_<serial>.png`
  - 封面：`workspace/base_image/<tag>/<MMDD>_<serial>_cover_01.png`

## 4) 现在到底用的什么大模型？如何切换？

结论要分两层：

1. 上传脚本层：
- **本仓库上传脚本内置模型 = 无**
- 上传时仍然只吃 `generation_map.json` 和本地图片

2. 统一控制台层：
- 现在可以在 `dashboard.py -> 提示词` 里维护“计划使用的模型”
- 这些字段会保存到 `config/prompt_studio.json`
- 但它不会替你直接发 API 请求，仍然是给你或后续外部生成链路使用

- 标题/简介来源：`generation_map.json`
- 缩略图来源：本地图片文件

所以“换模型”只能在你的外部生成链路做：

1. 外部脚本改模型（例如 GPT/Gemini/Claude/Flux/SDXL）
2. 产出文案与图片
3. 写回本仓库目录（`generation_map.json` + 图片文件）
4. 再跑渲染/上传

## 5) 分组（如“面壁者”）与风格/受众/语言/禁词

现在已经比之前更进一步：

- 分组映射位置：`config/upload_config.json`
- 风格/受众/语言/主提示词/标题库/模型模板：建议在 `dashboard.py -> 提示词` 按分组绑定
- 当天实际标题/简介/封面：在 `dashboard.py -> 当日内容` 改
- 保存当天内容后建议同步 `upload_manifest.json`，避免上传还吃旧标题

## 6) 常用命令（你当前这套）

```bash
# 渲染（只渲染，不上传）
py -3 daily_scheduler.py --standard 0312 --tags=面壁者 --render-only --song-count=1 --style=wave --particle=dust_bokeh --particle-opacity=0.55 --letterbox --zoom=slow --color-spectrum=WhiteGold --color-timeline=WhiteGold --film-grain=8 --color-tint=warm

# 上传（单频道）
py -3 batch_upload.py --tag 面壁者 --date 3.12 --channel 90 --auto-confirm --auto-close-browser

# 上传（多赛道批量）
py -3 bulk_upload.py --date 3.12 --tags 面壁者,芝加哥慢蓝调,烟嗓女伶爵士 --auto-confirm --auto-close-browser
```

## 7) 多赛道目录怎么放

如果你今天有多个赛道，推荐用统一结构：

- 音频：`workspace/music/<tag>/`
- 底图：`workspace/base_image/<tag>/`
- 输出：`workspace/AutoTask/<MMDD>_<tag>/`

例如今天要跑 5 个赛道，就准备 5 组目录：

- `workspace/music/赛道A/`
- `workspace/music/赛道B/`
- `workspace/music/赛道C/`
- `workspace/music/赛道D/`
- `workspace/music/赛道E/`

对应底图和输出目录也同理。

现在在 `dashboard.py` 里可以直接：

1. 在“快捷开始”把 tag 每行填一个
2. 点“批量建目录”
3. 去“路径配置”查看多赛道路径预览
4. 把素材分别丢进每个赛道目录
5. 回来执行渲染或批量上传
