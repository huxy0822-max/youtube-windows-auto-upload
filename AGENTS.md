# Repository Preferences

- Unless the user explicitly says otherwise, create GitHub repositories as public repositories.
- If a repository was created as private by default and the user gave no privacy requirement, change it to public.
- Even when the default is public, do not commit secrets, tokens, runtime databases, personal data, or other sensitive files. Sanitize or exclude them first.
- Unless the user explicitly says not to, sync each stable local version of this project to the GitHub repository after the fixes are finished.
